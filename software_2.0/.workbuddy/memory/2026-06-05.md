# 2026-06-05

## 智能导览人应用调试

**问题**: 智能导览人应用（`software-main/software-main/software`）加载不出来，白屏/3D 数字人不显示。

**项目信息**: UniApp + Vue 3 + TypeScript + Three.js + @pixiv/three-vrm，AI数字人景区导览系统。

### 发现的 4 个 Bug 及修复

1. **CRITICAL - profileStage 参数错误** (`src/components/DigitalAvatar3D.vue`)
   - `deepProfiler.profileStage()` 所有调用只传了 3 个参数 (id, name, fn)
   - 方法签名需要 4+ 个参数 (id, name, description, fn, critical?, dependencies?)
   - 导致 `fn` 为 undefined，`await fn()` 抛出 TypeError，初始化流程崩溃
   - 修复：所有调用处添加第 3 个参数（description 字符串）

2. **CRITICAL - Worker ArrayBuffer 转移导致数据丢失** (`src/utils/modelWorker.ts`)
   - `postMessage({...}, [arrayBufferData.buffer])` 使用 transfer list 转移了所有权
   - 主线程的 arrayBuffer 变为零字节空缓冲区
   - 之后 `loadVRMModel(arrayBuffer)` 收到空数据，GLTFLoader 解析失败
   - 修复：改用 `Array.from()` 复制数据，移除 transfer list

3. **VRM 文件下载超时** (`src/utils/tieredResourceLoader.ts`)
   - VRM 模型文件 98MB，默认超时 30 秒不够
   - 慢速网络下会超过超时限制导致加载失败
   - 修复：timeout 从 30000ms 增加到 180000ms (3分钟)

4. **SCSS @import 弃用警告** - 仅为弃用警告，不影响功能。暂不处理（改用 @use 会破坏所有 Vue 文件的变量引用）。

5. **CRITICAL - 构建后 Three.js/VRM 全局变量丢失** (`index.html` + `src/main.ts`)
   - Vite build 会删除 `index.html` 中的 `<script type="module">` 内联脚本
   - 导致 `window.THREE`、`window.VRMLoaderPlugin` 等全局变量在构建版本中不存在
   - 3D 组件代码依赖这些全局变量，全部失效
   - 修复：创建 `src/utils/threeBootstrap.ts`，用动态 `import()` 从 CDN 加载，在 `main.ts` 最前面执行；同时给 import map 添加 `@pixiv/three-vrm` 映射条目

6. **VRM 模型文件名含中文+空格导致 HTTP 404** (`DigitalAvatar3D.vue` 等)
   - 原文件名：`puppy护士 1 侧边灰 arkit校正.vrm`（含中文+空格）
   - 浏览器 fetch() 时 URL 编码后，Python http.server 无法正确匹配文件路径
   - 修复：重命名为 `guide_avatar.vrm`，更新所有引用路径（DigitalAvatar3D.vue、vrmModelConfig.ts、AppPreloader.vue、vrmModelValidator.ts）

7. **新增轻量2D CSS角色 `SimpleAvatar2D.vue`**
   - 3D/WebGL 在用户环境中始终无法创建上下文，改为纯 CSS 绘制 2D 卡通角色
   - 导游"小乐"形象：紫色长发 + 马尾 + 呆毛 + 粉色发饰 + 白色制服 + 紫色腰带 + "导"字徽章
   - 五官精致：渐变皮肤、带高光瞳孔、眉毛、鼻子、腮红
   - 5种表情：自然/开心/惊讶/难过/思考（每 3-6 秒自动眨眼）
   - 动画：呼吸浮动、说话弹跳、呆毛摇摆
   - 访客首页和导览对话页已切换为此组件
   - 原 VRM 模型文件 `guide_avatar.vrm` 保留未删除
